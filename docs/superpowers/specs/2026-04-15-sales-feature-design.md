# 销售功能设计文档

## 概述

为销售员提供POS界面，支持快速创建销售订单。订单状态变为"已完成"时自动扣减库存。

## 需求确认

| 需求项 | 确认内容 |
|--------|----------|
| 数据表 | 使用现有 sales_orders |
| 库存扣减 | 订单状态变为"已完成"时扣减 |
| 权限控制 | 销售员可查看全部订单，仅能操作自己创建的 |
| 界面功能 | 基本销售流程、快速搜索商品、库存实时显示、条码扫描 |
| 支付信息 | 不记录 |

## 方案选择

选择方案A：新建独立POS页面（/pos），与现有订单管理页面分离。

理由：
- 界面专为销售场景优化，操作效率高
- 条码扫描、快速搜索等POS特性体验更好
- 权限控制清晰：销售员只看POS页，管理员看订单管理

## 后端改动

### 1. SalesOrderService 扩展

文件：`server/src/modules/sales/sales-order.service.ts`

新增方法：
- `updateStatus(id, status, userId)` - 更新订单状态

库存扣减逻辑：
- 状态变为 `completed` 时触发
- 使用事务保证原子性
- 流程：检查库存 → 扣减库存 → 记录 inventory_log → 更新状态

### 2. Controller 权限控制

文件：`server/src/modules/sales/sales-order.controller.ts`

新增接口：
- `PUT /sales-orders/:id/status` - 更新订单状态

权限规则：
| 操作 | 权限 |
|------|------|
| 创建订单 | 所有登录用户 |
| 更新状态 | 订单创建者或管理员 |
| 删除订单 | 订单创建者或管理员 |
| 查看订单列表 | 所有登录用户 |

### 3. 库存扣减流程

```
订单状态 → completed
    │
    ▼
检查每个商品的库存是否充足
    │
    ├─ 不足 → 抛出异常，事务回滚
    │
    └─ 充足 → 扣减库存 + 记录日志(inventory_log, type: 'out')
```

## 数据流与错误处理

### 订单创建流程

```
用户填写订单信息
    │
    ▼
点击"提交订单"
    │
    ▼
前端校验：客户必填、商品数量 > 0
    │
    ├─ 失败 → 提示错误
    │
    └─ 成功 → 调用 POST /sales-orders (status=draft)
              │
              ▼
          返回订单ID，跳转到订单详情
```

### 订单完成流程

```
在订单管理页面点击"完成订单"
    │
    ▼
调用 PUT /sales-orders/:id/status { status: "completed" }
    │
    ▼
后端事务处理：
  1. 检查库存是否充足
  2. 扣减库存
  3. 记录 inventory_log
  4. 更新订单状态
    │
    ▼
返回成功
```
